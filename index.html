这是一份可分享的搭建指南：不包含任何真实服务器密码、代理密码或个人节点地址。照着做可以搭出一条“本地客户端 -> 首段 VPS -> 出口节点”的双跳链路。

目标架构#

第一跳负责“接入和分流”，第二跳负责“最终落地”。普通流量可以本地直连或从第一跳直连出去，只有需要指定落地地区的域名才转发到第二跳。对 AI/编程域名建议采用失败关闭策略：规则显式命中双跳，代理组里不放 DIRECT。

节点怎么选#

第一跳选型#

第一跳不一定要在目标网站所在国家，它更像“入口加速器”。选择标准：

  • 离自己近：大陆用户常见选择是新加坡、日本、香港、韩国等。
  • UDP 质量好：Hysteria2 走 QUIC/UDP，运营商到机房的 UDP 丢包越少越好。
  • 公网 IP 稳定：避免频繁换 IP，否则客户端和第二跳白名单都要改。
  • 云厂商安全组好管理：必须能放行自定义 UDP 端口。
  • 带宽够用：第一跳会承接所有客户端入口流量。

不要只看延迟最低。更重要的是高峰期丢包、绕路、晚间稳定性。

第一跳资源基线#

第一跳不要只看延迟和带宽，也要看内存和云盘 IOPS。Hysteria2 + sing-box 本身不重，但低配节点在连接抖动、DNS 查询、系统更新或 page cache 被回收时，可能出现“CPU 不高但整机假死”的状态。

建议基线：

  • 内存至少 1G，并配置 1G swap；预算允许时选 2G 内存更稳。
  • 云盘不只看容量，还要看 IOPS；低 IOPS 云盘在内存压力下容易被读 IO 风暴拖死。
  • 长期运行时 sing-box 日志级别用 warn，排障时再临时切到 info。
  • 看到 systemd-journald: Under memory pressure, flushing caches.、load average 很高、blocked 很多、sar -d 里 rkB/s 高而 wkB/s 很低时，优先按内存压力和读 IO 风暴排查，不要误判为日志写爆。

第二跳选型#

第二跳才决定目标网站看到的出口 IP。选择标准：

  • 地区匹配目标：如果目标服务希望美国环境，第二跳就选美国。
  • IP 类型干净：AI、风控敏感站点通常更偏好住宅/ISP/干净静态 IP。
  • 支持账号密码：SOCKS5 必须有认证。
  • 支持白名单：只允许第一跳 IP 访问，防止出口被扫到后被滥用。
  • 尽量静态：频繁换 IP 会影响账号风控、登录状态和可用性。

如果第二跳是自有 VPS，可以直接在第二台机器上跑 SOCKS5；如果是第三方住宅代理，则在第一跳配置 SOCKS5 出站即可。

节点地址怎么选#

  • 客户端 server 填第一跳地址，不填第二跳地址。
  • 有自己的域名时，推荐把域名 A 记录指向第一跳，并申请真实 TLS 证书。
  • 没有域名时，也可以用第一跳 IP + 自签证书，但客户端要开启跳过证书验证。
  • 第二跳地址只写在第一跳的 sing-box 配置里，不暴露给本地客户端。
  • 第二跳如果是自建 SOCKS5，防火墙只放行第一跳 IP。

端口建议随机高位，不建议用太常见的 443、8443、1080。随机高位端口不是安全措施本身，但能少一些无意义扫描。

常见问题#

客户端连接不上第一跳#

  • 检查第一跳安全组是否放行 UDP <FIRST_PORT>。
  • 检查第一跳 systemctl status sing-box。
  • 检查 Clash 配置里的 IP、端口、密码、SNI 是否一致。
  • 自签证书场景下确认 skip-cert-verify: true。

第一跳能连，但 AI 没走第二跳#

  • 检查第一跳配置里的 domain_suffix 和 domain_keyword。
  • 检查第二跳 SOCKS5 地址、账号、密码。
  • 检查第二跳防火墙是否放行 <FIRST_IP>。
  • 检查是否被 QUIC/UDP 绕过，必要时继续保留 AI 域名 UDP block 规则。
